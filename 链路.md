## 📊 **MedLabAgent 完整转发链路**

### **第一部分：文字消息转发链路**

```
用户输入 → 前端发送 → Java 后端流处理 → LangChain Agent → 流式返回 → 前端展示
```

#### **1️⃣ 前端：用户输入文本**

**文件：ChatWindow.vue**

- 用户在输入框输入文本，点击"发送"或按 Enter
- `sendMessage()` 函数：
  - 将用户消息添加到 Pinia Store
  - 创建空的助手消息对象
  - 调用 `ApiService.streamChat()`

#### **2️⃣ 前端：发起流式请求**

**文件：ApiService.js**

- `streamChat()` 方法：
  - 使用 Fetch API 发起 **POST** 请求到 `/api/v1/agent/chat/stream`
  - 设置 `Accept: text/event-stream` 请求头
  - 获取响应的 `ReadableStream`，逐块解析 SSE 事件
  - 实时处理 `[META:{...}]` 元数据（疾病诊断标记）
  - 流式更新聊天消息内容

#### **3️⃣ Java 后端：接收流式请求**

**文件：AgentController.java**

- `/api/v1/agent/chat/stream` 端点
- 从 JWT Token 中提取 `userId`
- 创建 `SseEmitter` 对象返回给前端
- 在后台异步处理对话

#### **4️⃣ Java 后端：异步调用 LangChain 服务**

**文件：SessionChatService.java**

- 组装用户医学历史（过敏信息、既往病历）
- 使用 WebClient 异步调用 LangChain 服务
- 监听响应流，逐块读取数据
- 通过 `SseEmitter.send()` 实时推送给前端
- 保存每轮对话到数据库 (SessionMessage 表)(需要修改)

#### **5️⃣ Python LangChain 服务处理**

**文件：main.py**（配置）

- 接收用户查询
- 检索医学知识库（向量数据库）
- 调用 LLM（GPT-4o/Claude/Qwen 等）
- **流式返回** 分析结果
- 附加 `[META|diseases:...,drugAllergies:...]` 标签

#### **6️⃣ 前端：接收元数据并提取**

**文件：ChatWindow.vue**

- `onDone()` 回调处理元数据
- 从 AI 回复中提取：
  - `isMedical`: 是否是医学诊断
  - `extractedDiseases`: 疾病列表
  - `extractedDrugAllergy`: 过敏药物
- 展示"记录到病历"确认按钮

### **第二部分：图片/文件转发链路**

```
用户选择文件 → 前端上传 → Java保存文件 → 自动预取OCR → 异步识别 → 自动发送分析
```

#### **1️⃣ 前端：用户选择文件**

**文件：ChatWindow.vue**

- 用户点击"📤 上传"按钮
- `uploadFile()` 触发文件选择器
- `handleFileUpload()` 处理选中的文件

#### **2️⃣ 前端：上传文件**

**文件：ApiService.js**

- `uploadReport(file)` 方法：
  - 创建 `FormData` 对象
  - 发送 **POST** 请求到 `/api/v1/file/upload-report`
  - 返回文件名、文件路径

#### **3️⃣ Java 后端：接收并保存文件**

**文件：FileController.java**

- `/api/v1/file/upload-report` 端点
- 验证文件大小、类型（jpg/png/pdf 等）
- 生成 UUID 作为文件名：`{UUID}-{originalName}`
- 保存到 `${medlab.upload.dir}` 目录
- 返回浏览器 URL 和 OCR 访问 URL

#### **4️⃣ Java 后端：预取 OCR 缓存（异步）**

**文件：VisionService.java**

- `prefetchOcr()` 方法：
  - 后台异步线程执行
  - 立即发起 OCR 识别请求（**不阻塞** 上传响应）
  - 结果缓存在 Redis 或内存中

#### **5️⃣ Java 后端：异步 OCR 识别**

**文件：OcrServiceClient.java**

- `analyzeVisionAsync()` 方法：
  - 通过 WebClient 调用 Python OCR 服务
  - 使用 Reactor Mono 反应式编程（非阻塞）
  - 支持超时控制（90 秒）
  - 返回 `AnalyzeVisionResponse` 对象（结构化数据）

#### **6️⃣ Python OCR 服务：识别文件**

**文件：main.py**

- 接收文件路径（HTTP URL 形式）
- 下载图片文件到本地 `/app/uploads`
- 调用 LLM Vision API：
  - **Qwen VL-Plus** (默认)
  - **GPT-4o Vision**
  - **Claude 3.5 Vision**
  - 支持自定义模型选择
- 解析 LLM 响应为结构化 JSON：
  ```json
  [
    {
      "item": "红细胞计数",
      "value": "4.5",
      "unit": "10¹²/L",
      "normal_range": "4.0-5.5",
      "status": "正常"
    }
  ]
  ```
- **缓存到 Redis**（避免重复识别）

#### **7️⃣ 前端：自动合成分析请求**

**文件：ChatWindow.vue**

```javascript
// 上传成功后自动构造消息
userInput.value = `请分析这张化验单：${response.filePath}`;
await sendMessage(); // 立即发送到 LangChain
```

- 自动填充输入框：`请分析这张化验单：/uploads/{uuid}-report.jpg`
- 立即调用 `sendMessage()`，触发 LangChain 分析流程
- **用户无需手动和 AI 交互**

#### **8️⃣ LangChain 分析结果**

**文件：main.py**

- 接收："请分析这张化验单：/uploads/xxx.jpg"
- 联网获取 OCR 识别结果
- 结合医学知识库进行诊断分析
- 生成医学建议

#### **9️⃣ 前端：显示确认按钮**

- AI 回复完成后，展示"✅ 记录到病历"按钮
- 用户可确认是否保存到医疗历史

### **⚡ 关键优化设计**

1. **异步非阻塞**：文件上传后立即返回，OCR 在后台进行（`CompletableFuture`）
2. **流式推送**：LangChain 回复不用等全部生成，实时推送给前端
3. **缓存加速**：Redis 缓存 OCR 结果，避免重复识别
4. **自动触发**：上传文件后自动向 AI 发送分析请求，0 手动操作
5. **WebClient**：使用 Reactor 异步 HTTP 客户端，非 RestTemplate 同步阻塞
6. **JWT 认证**：所有请求都验证 Token，绑定用户身份
