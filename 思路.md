## GAT-ReAct 协同会诊方法：基于图注意力引导的科室(工具)动态调用

### 核心思想

将图注意力网络（GAT）嵌入 ReAct 智能体流程，利用**指标生理关联图**和**专家协作图**两重图结构，动态计算当前患者状态下各科室的**调用概率**。该概率综合考虑：

- **科室意见的参考性**：由 GAT 在指标关联图上计算出的关键指标簇与科室的匹配强度决定；
- **科室间的关联性**：由专家协作图中定义的协作关系（如协同、前置、互斥）进一步调制。

最终，LLM 在每轮推理中不再盲目选择科室工具，而是在 GAT 输出的概率分布引导下进行决策，实现精准、可解释的多科室会诊。

---

### 整体架构

```
[患者数据] → [GAT 图推理] → [科室调用概率分布] → [ReAct 循环] → [科室工具调用] → [结果反馈] → (下一轮)
```

---

### 1. 图结构构建（离线阶段）

#### 图 A：指标生理关联图

- **节点**：化验指标（Cr, BUN, WBC, ALT, …）
- **边**：生理耦合关系（正相关、同系统、临床路径），边权重由医学知识确定。
- **作用**：反映指标间的内在联系，用于计算当前异常指标簇。

#### 图 B：专家协作图

- **节点**：科室 Agent（肾内科、血液科、心内科、感染科、检验科等）
- **边**：
  - `COLLABORATE`：常需联合诊疗的科室（如肾内科与心内科），带权重表示协作紧密度。
  - `PRECEDES`：调用顺序依赖（如检验科应优先于肾内科）。
  - `MUTUAL_EXCLUSIVE`：互斥科室（避免同时调用）。
- **作用**：刻画科室间的协作模式，用于调制调用概率。

---

### 2. ReAct 流程中的 GAT 引导

每轮循环包含以下步骤：

#### Step 1：GAT 前向传播——生成科室调用概率分布

**输入**：

- 当前患者化验指标值（归一化）、异常标志、参考范围；
- 患者元数据（年龄、性别、主诉嵌入）；
- 上一轮 Observation 文本嵌入（可选）。

**处理流程**：

1. **在指标关联图（图 A）上运行 GAT**
   - 节点特征向量由指标值和元数据拼接而成。
   - GAT 计算各指标间的注意力系数，输出更新后的指标节点表示，并识别出**关键指标簇**（例如 Cr、BUN、UA 聚类权重高）。
   - 同时得到每个指标的“当前重要性权重”。

2. **映射到科室初始调用概率**
   - 通过预设的“指标–科室”映射表（如 Cr 映射至肾内科），将关键指标簇的重要性权重聚合到各科室上，得到**初始调用概率** $p_0(d)$。
   - 该概率反映了科室意见的**参考性**：指标越重要且与该科室关联越强，则初始概率越高。

3. **在专家协作图（图 B）上运行第二层 GAT**
   - 节点为科室，节点特征为初始调用概率 $p_0(d)$（或结合历史调用信息）。
   - 边为协作关系，GAT 计算科室间的注意力系数，根据协作紧密度对初始概率进行**调制**，输出**最终调用概率** $p(d)$。
   - 例如：若肾内科与心内科存在强协作关系，当肾内科初始概率高时，心内科的最终概率也会相应提升；若存在互斥关系，则概率会压低。

**输出**：

- 科室调用概率分布 $\{p(d)\}$，可设定阈值仅保留 Top‑K 候选科室。
- 关键指标簇及其注意力权重（用于可解释性）。

#### Step 2：Thought（LLM 生成）

- 将 GAT 输出的概率分布作为系统提示的一部分注入，例如：
  ```
  当前 GAT 分析结果：
  - 关键指标簇：Cr、BUN、UA（权重 0.85）
  - 建议科室调用概率：肾内科(0.72)、心内科(0.45)、检验科(0.38)
  请根据以上信息和患者数据，生成下一步推理思考。
  ```
- LLM 在生成 Thought 时，既可利用自身知识，又受到概率分布的约束，避免偏离高概率科室。

#### Step 3：Action（科室工具调用）

- **科室选择**：可由 LLM 从概率分布中自主选择（通常选最高概率），也可由系统强制选择概率最高的科室以降低随机性。
- **执行调用**：调用对应科室 Agent 的工具接口（如查询专科知识库、获取诊疗建议等），返回结构化结果。

#### Step 4：Observation（结果吸收与状态更新）

- 将科室返回的建议、检查结果等转化为文本或结构化数据，作为 Observation。
- 更新患者状态（例如新增检验结果或诊断线索），并将 Observation 嵌入下一轮 GAT 输入中，实现闭环。

---

### 3. 技术亮点

- **动态概率调制**：科室调用概率不再固定，而是随患者数据实时变化，且通过专家协作图融入科室间关联，使决策更符合真实临床协作习惯。
- **参考性与协作性并重**：初始概率体现科室意见的“参考价值”（基于指标关联），最终概率进一步体现“协作必要性”（基于科室间关联），两者结合使 Agent 在多科室会诊中更加智能。
- **可解释性**：可向用户展示关键指标簇及其权重，以及科室概率的调制路径（如“因肾内科概率高且与心内科协作紧密，心内科概率提升”），增强医生信任。

---

### 5. 方法定位与价值

本方法将图注意力机制引入智能体决策流程，使 LLM 在多科室会诊中实现**有据可依的动态调用**，既保留了 LLM 的泛化能力，又通过知识图谱约束提升了安全性和准确性。这一神经符号融合的思路，特别适用于医疗等高风险领域，具有明确的临床转化潜力和学术创新价值。
